Food Truck Finder Website Prompt

This is the same prompt captured in website-build-prompt.md, but formatted so it opens cleanly from the live site. 

Create a polished, conversion-focused marketing website for Food Truck Finder inside this GitHub Pages repo.

Important context:
- Use plain HTML, CSS, and lightweight JavaScript.
- No build step, framework, or package manager.
- The site must serve foodies, food truck owners, and event organizers.
- The visual direction should feel warm, energetic, and local-food-driven.
- Keep privacy policy, delete-account, download links, and support email visible.

Required sections:
1. Hero with layered screenshots and clear product explanation
2. Three-role overview for foodies, owners, and organizers
3. Demo section with embedded video or polished placeholder
4. Foodies section with how-to and screenshots
5. Owners section with business outcomes, onboarding steps, and screenshots
6. Organizers section with discovery and coordination value
7. Feature clarity section in plain language
8. FAQ
9. Final CTA
10. Footer with legal, support, media, and prompt links

Available assets live in:
- assets/media/foodies/
- assets/media/owners/
- assets/media/organizers/
- assets/media/video/

Voice:
- Direct
- Useful
- Credible
- No startup fluff
- Focus on practical value and real usage

Technical rules:
- Semantic HTML
- Responsive navigation
- Centralized CSS variables
- Minimal JavaScript
- Screenshots must scale cleanly on mobile
- If ftf-demo.mp4 is missing, show a graceful video placeholder instead of a broken player
